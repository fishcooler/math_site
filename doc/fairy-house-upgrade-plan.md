# 童话屋养成与贴纸兑换升级方案

## 1. 背景与目标

在现有数学练习积分体系上，增加一条轻量养成副线：

- 积分用于兑换贴纸
- 贴纸可收集、可展示
- 童话屋场景中的物品随贴纸逐步解锁
- 每个物品可提供“童话解释 + 数学知识解释”

核心原则：**不破坏原有练题与积分主链路**，只在积分消费与展示层扩展。

## 2. 设计原则

- 保持学习主线稳定：练题 → 得积分 → 查错题 → 排行榜
- 贴纸只做展示与养成，不改变题目难度
- 童话屋解释以教育延伸为主，不是纯装饰
- 先做静态闭环，再做动效与进阶养成
- 数据结构尽量拆表，不在 `User` 表堆字段

## 3. 产品形态建议

### 3.1 新增页面

- `/fairy-house`：童话屋主场景
- `/stickers`：贴纸图鉴 / 商店 / 背包（可先合并成一页）

### 3.2 童话屋结构

固定一版主屋场景，例如：

- 屋顶星星
- 门与窗户
- 书架
- 壁炉
- 地毯
- 花园
- 桌椅
- 苹果树

每个位置绑定一个可解锁物品，并支持点击弹出解释卡：

- 物品名称
- 解锁方式
- 数学主题解释
- 童话化解释

## 4. 积分与贴纸耦合方式

### 4.1 积分商店

- 积分用于购买“贴纸包”或单张贴纸
- 建议初期只做单张贴纸兑换，降低实现复杂度

### 4.2 贴纸主题

- 计算系列：加法小火苗、减法雨滴
- 规律系列：彩虹阶梯、螺旋魔杖
- 比较系列：天平徽章
- 应用题系列：小背包、苹果篮
- 坚持系列：星星徽章、勇气披风

### 4.3 奖励边界

- 贴纸只负责收集、展示与解释
- 练习成绩奖励仍由现有 `PointSystem` 控制
- 贴纸兑换只消费积分，不影响成绩统计

## 5. 轻养成机制建议

不建议做战斗养成，建议做“房屋养成”：

- 随贴纸增加，小屋逐渐变完整
- 可设置轻量成长属性：
  - 聪明值：来自计算相关贴纸
  - 勇气值：来自坚持练习相关贴纸
  - 耐心值：来自错题复习相关贴纸

这样避免复杂系统，但比单纯收集更有“养成感”。

## 6. 数据模型建议

### 6.1 `Sticker`

- `id`
- `name`
- `icon`
- `series`
- `price`
- `growth_tag`（对应聪明/勇气/耐心等）
- `description`

### 6.2 `UserSticker`

- `id`
- `user_id`
- `sticker_id`
- `obtained_at`

### 6.3 `FairyRoomItem`

- `id`
- `name`
- `icon`
- `unlock_condition_type`
- `unlock_condition_value`
- `story_description`
- `math_description`

### 6.4 `UserFairyRoom`

- `id`
- `user_id`
- `room_item_id`
- `unlocked_at`

## 7. API 建议

### 7.1 贴纸系统

- `GET /api/stickers`：获取贴纸图鉴与状态
- `POST /api/exchange-sticker`：积分兑换贴纸
- `GET /api/user/stickers`：获取用户已拥有贴纸

### 7.2 童话屋系统

- `GET /api/fairy-house`：返回房间解锁状态
- `GET /api/fairy-house/item/{item_id}`：返回某物品解释卡

## 8. 页面交互建议

### 8.1 贴纸页

- 顶部：用户名 / 当前积分
- 中部：贴纸图鉴（已拥有 / 未拥有）
- 右侧或底部：兑换区

### 8.2 童话屋页

- 顶部：收集进度 / 轻养成数值
- 中部：房间主场景
- 点击物品弹出解释卡
- 未解锁物品显示剪影

## 9. 实现顺序建议

### 第一阶段：贴纸定义 + 积分兑换 + 收藏页

目标：打通最小闭环

- 新建贴纸表与用户贴纸表
- 新增贴纸图鉴接口
- 新增积分兑换接口
- 新增前端页面入口与交互

### 第二阶段：童话屋场景 + 物品解锁 + 解释卡

目标：让贴纸有展示舞台

- 新建房间物品表
- 新建用户房间解锁表
- 实现点击解释卡
- 绑定收集进度

### 第三阶段：轻养成属性 + 房屋升级动效

目标：增强养成感

- 增加聪明/勇气/耐心等成长值
- 增加阶段性房屋外观变化
- 增加解锁动画

### 第四阶段：家长与教学说明

目标：增强产品教育说服力

- 家长看板增加贴纸与养成说明
- 解释“贴纸背后的数学学习意义”

## 10. 当前建议结论

优先做：

1. 积分兑换贴纸
2. 贴纸收藏页
3. 童话屋静态场景
4. 物品解释卡

这样可以最快验证价值闭环，同时避免对现有数学练习系统造成过大改动。
