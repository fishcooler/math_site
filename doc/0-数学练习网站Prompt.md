# Prompt：幼小衔接数学练习网站

## 角色定义

你是一位全栈开发工程师兼教育内容设计师，擅长为儿童设计有趣、易用的学习工具。

## 任务目标

为一个正在做"幼小衔接"阶段练习的小朋友，开发一套**完整的数学练习网站**。网站需要具备用户管理、题目练习、自动批改、错题记录等功能。

## 一、数学试卷内容要求

### 1. 目标受众
- 幼小衔接阶段的小朋友（约5-7岁）
- 已学过小学上学期100以内的加减法

### 2. 题目结构（共30题）

| 题型 | 题号 | 数量 | 分值 | 说明 |
|------|------|------|------|------|
| 直接计算（简单） | 1-10 | 10题 | 每题3分 | 10以内/20以内加减法 |
| 填空计算（进阶） | 11-20 | 10题 | 每题3分 | 100以内加减法 |
| 找规律填数 | 21-23 | 3题 | 每题4分 | 等差数列（+2/+3/+5/-10等） |
| 比较大小 | 24-25 | 2题 | 每题4分 | 在○里填">"、"<"或"=" |
| 数一数 | 26-27 | 2题 | 每题4分 | 数图形个数（三角形/圆形/正方形/五角星） |
| 解决问题（应用题） | 28-30 | 3题 | 每题4分 | 生活场景应用题，含步骤分解引导 |

### 3. 题目生成规则
- **计算题**：随机生成，每次刷新不同
- **找规律题**：从预设规律库中随机抽取（如 +2、+3、+5、-10、奇数等）
- **比较大小题**：随机生成算式与数字的比较
- **数一数题**：随机选择图形类型和数量（3-8个）
- **应用题**：从模板库中随机生成（购物、动物、教室等生活场景）

### 4. 应用题步骤分解引导
每道应用题需要展示分步计算的引导框，例如：
```
小明有12颗糖，又买了8颗，现在有多少颗糖？
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 第一步    │  │ 第二步    │  │ 第三步    │
│ 有12颗   │  │ 又买8颗  │  │ 12+8=___ │
└──────────┘  └──────────┘  └──────────┘
答：现在有____颗糖。
```

## 二、视觉设计要求

### 1. 整体风格
- 参考**儿童数学练习册**风格（如学而思、猿辅导等）
- 色彩丰富、活泼有趣，适合小朋友使用
- 柔和的配色方案（浅蓝、浅粉、浅绿、浅黄等）

### 2. 页面元素
- 题目以**卡片形式**呈现，带彩色背景
- 每题配有可爱的**小图标**（小动物、植物、表情等）
- 比较大小题使用**下拉选择框**（>、<、=）
- 其他题目使用**输入框**

### 3. 批改结果展示
- 正确：绿色边框 + ✅ 标记
- 错误：红色边框 + ❌ 标记 + 显示正确答案
- 提交后弹出**成绩弹窗**，显示分数和鼓励语

## 三、功能要求

### 1. 用户管理
- 支持多个小朋友独立账号
- 输入**用户名**（英文/数字）和**小朋友姓名**即可登录
- 首次登录自动注册，再次登录自动识别
- 不同小朋友的答题记录和错题本互相独立

### 2. 练习功能
- 页面加载时自动生成30道题
- 顶部显示**进度条**（已答题数/总题数）
- 支持**重新生成**按钮，一键刷新所有题目
- 题目按类型分组展示（计算题、找规律、比较大小、数一数、应用题）

### 3. 自动批改
- 点击"提交答案"后自动判分
- 逐题标注对错
- 弹窗显示总分和鼓励语：
  - 90分以上："🌟 太棒了！继续保持！"
  - 70-89分："👍 不错哦，再努力一点！"
  - 70分以下："💪 加油！多练习就会进步的！"

### 4. 错题本
- 答错的题目自动记录到错题本
- 每条错题记录包含：
  - 错误答案 vs 正确答案
  - 解题说明/解析
  - 记录时间
- 支持**标记已复习**和**删除**操作
- 顶部统计卡片显示：总错题数、已复习、待复习
- 无错题时显示鼓励页面"🎉 太棒了！你还没有错题"

## 四、技术要求

### 1. 技术栈
- **后端**：Python Flask
- **数据库**：SQLite（通过 Flask-SQLAlchemy）
- **前端**：HTML + CSS + JavaScript（原生，无需框架）
- **字体**：Microsoft YaHei / PingFang SC

### 2. 数据库模型
```
User（用户表）
├── id, username, name, created_at

Question（题目表）
├── id, question_type, content, answer, options, explanation, difficulty

Answer（答题记录表）
├── id, user_id, question_id, user_answer, is_correct, answered_at, practice_set

Mistake（错题表）
├── id, user_id, question_id, wrong_answer, correct_answer, explanation, created_at, reviewed
```

### 3. 页面结构
```
/                    → 登录页（index.html）
/practice            → 练习页（practice.html）
/mistakes            → 错题本页（mistakes.html）
/api/generate_questions  → 生成题目API
/api/submit_all          → 提交答案API
/api/mark_reviewed/<id>  → 标记已复习API
/api/delete_mistake/<id> → 删除错题API
```

### 4. 响应式设计
- 支持桌面和平板访问
- 题目卡片网格布局（auto-fill, minmax(280px, 1fr)）

## 五、交付物

1. **完整项目代码**（可直接运行）
2. **一键启动脚本**（启动.bat，自动安装依赖）
3. **README.md**（使用说明）
